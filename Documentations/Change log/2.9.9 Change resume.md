## Folder: inputs
* A new folder named `NetLogon` has been added. This folder is copied to `\\Domain\NetLogon`. Its content is then used by GPO to create schedule or deploy scripts with a dynamic reference to the configuration files.
* The *PolicyDefinitions* has been updated

## GPO SMBv1 Overhaul

A complete redesign of the SMBv1 GPOs has been implemented:
- A new script now supports multiple OS versions and PowerShell versions.
- It addresses common bugs such as:
  - Dependency issues causing Lanman Workstation service failures.
  - PowerShell V2 cmdlet compatibility.
  - System account limitations with COM component loading for DISM calls.

## 🔨 Fixes

✅ __GPO SMBv1__  
the script has been completly rewritten to fiw bugs and take into accounts OS ansd PowerShell differences.  

✅ __HAD-LogonCache-2__  
cache size increased to 3 to accommodate WHFB authentication, preventing admin cache overwrite.  

✅ __HAD-DMA-Protection__  
resolved conflicts between kernel DMA protection and BitLocker settings.  

✅ __LocalAdmin Scripts__  
updated to only use the custom.xml file and refering to the netlogon folder to get the proper version by eache DC. Some fixes done on depedent scripts too.

✅ __HAD-WebProxyAutoDiscovery-Diabled__  
the registry value now set the service to manual start instead of disabled.  

✅ __NBT-NS__  
A script will now force the NBT-NS service to be disabled on each network interface.  

✅ __UNC Hardened Paths__  
this GPO is now linked to the root and no more to the domain controllers OU.  

✅ __SSL and TLS GPO__  
the gpo has been reviewed and updated to match with new instruction from MS.
